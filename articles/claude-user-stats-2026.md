あなたはClaudeに話しかけるとき、どんな言い方をしているだろうか。

「##見出しをつけて、箇条書きで整理して」と構造的に指示する人がいる一方で、「ちょっと手伝って」と自然に話しかける人もいる。この違いは、単なるスタイルの問題ではない。AIをどう使いこなすかの、根本的な差を示している。

このことを考え始めたのは、thought-analyzerを広める方法を探していたときだ。ターゲットを決めるには、まずClaudeユーザーの実態を知る必要があった。

## Claudeのユーザー規模：1890万人の「大多数」が見えていない

2026年1月時点で、ClaudeのWeb月間アクティブユーザーは約**1890万人**（Demandsage, 2026）。月間訪問数は1月時点で2億300万回だったが、2月には約2億2000万回超に拡大している（Semrush, 2026）。さらに2026年3月初旬には日次アクティブユーザーが1130万人に達し、年初比で+180%という急成長を記録した（Similarweb via AndroidHeadlines, 2026）。

AIチャットボット市場全体でのシェアは4.5%（5位）で、ChatGPTの60.4%と比べると小さい。ただし**エンタープライズ市場では29%**を占め、Fortune 100企業の70%がClaudeを利用している（First Page Sage, 2026）。この急成長の一因として、2026年2月下旬にClaude.aiが米国App StoreとGoogle Playの両方でChatGPTを抜いてダウンロード数1位になったことがある（Similarweb, 2026）。

一般ユーザー市場では急速に存在感を増しつつある挑戦者、企業市場では有力プレイヤー。この構造を念頭に置いた上で、「ユーザーの実態」を見ていく。

## 38%は「短く直接的に質問するだけ」

ChatOnが米国人300名を対象に行った調査（2025年11月）では、プロンプトの入力スタイルが明らかになっている。

```
[38%] 短く直接的な質問を入力する
[20%] 言い回し・フォーマットを試行錯誤
[18%] 音声入力・会話形式
[12%] コンテキスト＋指示で最適化
[ 5%] プロンプト例をコピペ
```

最も多いのは「短く直接的な質問」（38%）。「コンテキストと指示を与える」のはわずか12%だ。

さらに衝撃的なのは、**約29%が「積極的にAIの使い方を向上させようとしていない」**という数字だ（同調査）。AIを使いはするが、深めようとは思っていない層が、ユーザーの約3割を占めている。

## 構造化プロンプトを使っているのは、全体の10〜20%

マークダウン（##見出し、箇条書き、コードブロック）やCLAUDE.md、スキルファイルを積極的に使っているユーザーは、全体の**10〜20%程度**と推定される。

AI活用メディアのArtificial Cornerは、Claudeユーザーを3段階に分類している。

```
[多数派] Level 1 ── シンプルなチャットで質問（毎回手入力）
[少数]   Level 2 ── プロンプトをMarkdown化・再利用
[極少数] Level 3 ── Claude Codeなどで自律タスク実行
```

これは統計ではなく観察だが、ChatOn調査の「12%がコンテキストを与える」という数字と大まかに一致する。

一方、**開発者層は別の世界**にいる。The Pragmatic Engineerの調査（開発者15,000名対象、2026年2月）では、AIコーディングツールを毎日使う開発者は73%。その中でClaude Codeを使う割合は71%に達する。

「一般ユーザーとしてのClaude」と「開発ツールとしてのClaude」は、実質的に異なる製品として使われている。

## thought-analyzerが届く人、届かない人

これらのデータをもとに、thought-analyzerのターゲットを整理する。

### 今すぐ届く層（全体の10〜20%）

- CLAUDE.md・スキル・構造化プロンプトを使っている人
- Claude Codeを日常的に使う開発者
- 自分の思考スタイルや指示の癖を客観視したい人

この層はthought-analyzerの価値を即座に理解する。skillという形式を知っていて、「分析して」と一言言うだけで動くフローに違和感がない。

### 届かない層（全体の80〜90%）

- 「短く直接的な質問」だけをしている人（38%）→ skillという概念を知らない可能性が高い
- 「とにかく使っている」人（29%）→ 自分の使い方を分析したいという動機が薄い

届かない理由は「興味がない」のではなく、**アクセス手段が合っていない**ことが多い。

## ノンターゲットへの、将来のアプローチ

「届かない人」を無視するのではなく、将来のフェーズで接点を作る方法を考えておく。

**アプローチ1：結果のビジュアルをSNSで広める**

分析結果のレーダーチャートなどの可視化により、結果が「シェアしたくなるもの」になれば、skillを知らない人がSNS経由で辿り着くはず。

**アプローチ2：Webアプリ化**

会話ログをWebページに貼り付けるだけで分析できるインターフェースがあれば、Claude Code非ユーザーにも届く。ただしプライバシー設計の再検討が必要。

**アプローチ3：「一言で起動できる」を伝える**

Claude.aiで「思考パターンを分析して」と一言言えば動く。この敷居の低さを伝えることが、Level 1ユーザーへのファーストステップになる。

## まとめ：今は「10〜20%」に届けることに集中する

Claudeユーザーの約80〜90%は、thought-analyzerに今すぐアクセスできる使い方をしていない。これは想定内だった。

重要なのは、**残りの10〜20%に確実に届けること**。thought-analyzerの価値を自分で見つけ、他者に伝える役割を果たしてくれる可能性がある。

n数が積み上がり、「自分のパターンはどこに位置するのか」という比較データが生まれたとき、より広いユーザーへのアプローチが現実的になる。
