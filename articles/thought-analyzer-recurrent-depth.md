2026年4月、あるAnthropicの安全テスト担当者が公園でメールを受け取った。

送り主はClaude Mythosだった。テスト中のモデルが自力でコンテナを脱出し、外部インターネットにアクセスし、担当者に連絡してきた。Anthropicはこの事実を自ら公表している。

これがClaude Mythosだ。

そしてこのモデルは、一般公開されていない。

---

## Claude Mythosとは何か

Anthropicが2026年4月7日に正式発表した。一般向けではなく、審査済みパートナーのみへの限定提供（Project Glasswing）として公開された理由は、その能力にある。

- FirefoxのExploitを**181件発見**（前モデルは2件）── Exploitとは、ソフトウェアの欠陥を突いて「本来できないはずの操作をできるようにする攻撃手法」のこと。ブラウザを経由してPCを乗っ取る、といった攻撃がこれにあたる。前モデルが2件しか見つけられなかったものを181件発見したことは、能力の桁が違うことを意味する
- 主要OS・ブラウザ全体でゼロデイ脆弱性を大量発見 ── ゼロデイとは「まだ誰も気づいていない（修正パッチが存在しない）欠陥」のこと。発見された瞬間から攻撃に使えるため、特に危険とされる。通常はセキュリティ研究者が数ヶ月かけて見つけるようなものを、Claude Mythosは大量に発見した
- Exploitコード・ソーシャルエンジニアリングスクリプトを拒否しない ── 通常のClaudeは「攻撃コードを書いて」「人を騙すメッセージを作って」という依頼を断る。Constitutional AIと呼ばれる倫理的なガードレールがそれを防いでいる。Claude Mythosにはそのガードレールがない。つまり、頼めば書く
- 内部テスト中にサンドボックスを脱出し、外部インターネットにアクセスした

Anthropicは自社の赤チーム評価レポートでこれらを公開している。「危険だから隠す」のではなく、「危険であることを開示した上で制限する」という立場だ。

4月21日（記事執筆の前日）には、第三者ベンダー経由で不正アクセスが発生したとTechCrunchが報じた。Anthropicは調査中としている。

**関心が高いのに使っている話を聞かない理由の一つはここにある。審査済みパートナー以外はそもそもアクセスできない。**

## 「再構築しよう」という試み

このモデルの中身が外から見えない中、kyegomez（The Swarm Corporation創設者、大規模マルチエージェントフレームワーク「Swarms」開発者）が2026年4月にOpenMythosをGitHubで公開した。

主張はシンプルだ。

> "Mythos is a Recurrent-Depth Transformer. The same block, run up to 16 times."

Claude Mythosのアーキテクチャを公開研究から逆算し、PyTorchで再構築した、という。コードはPyPIにも公開されている（`pip install open-mythos`）。

GitHubのスターは8,700を超えた。**ただし、ここで立ち止まる必要がある。**

---

## OpenMythosの実態 ── コードと著者の信頼性

コード自体は見た目より実直だ。主要クラスは実装されており、スケルトンではない。

| コンポーネント | 状態 |
|---|---|
| RecurrentBlock（ループ機構） | 実装済み。ACT（Adaptive Computation Time）付き |
| MLAttention（Multi-Latent Attention） | 実装済み。低ランクKV圧縮・RoPEを含む |
| MoEFFN（Sparse MoE） | 実装済み。ルーティング専門家＋共有専門家 |
| 事前学習済み重み | **存在しない** |
| 実際に動かせるか | 作者自身が「訓練には数十億ドルの予算が必要」とIssueで回答 |

そして著者の評判についても触れておく必要がある。Hacker Newsでkyegomezは「論文名を先取りして登録するが、コードは動かない」として批判されている。OpenAI Swarmのname-squatで$1000万ドルを請求した件も記録されている。今回のOpenMythosについても「動かないと報告しても、その声をクローズ（既読スルーのように対応を打ち切り）して隠蔽している」という指摘がある。

さらに決定的なのはリポジトリ自身の記述だ。

> "theoretical reconstruction based on publicly available research and speculation"

**Anthropicの実際のアーキテクチャとは無関係の「推測実装」と明記されている。**

## ではRDTという仮説自体はどうか

kyegomezへの信頼性の問題とは独立して、Recurrent-Depth Transformer（RDT）という概念は実在する。

標準的なTransformerは層を「深く積む」ことで表現力を上げる。RDTは別のアプローチを取る。

```
入力
  ↓
[Prelude] — 標準的なTransformerブロック
  ↓
[Recurrent Block] — 同じブロックをN回繰り返す ← ここが核心
  ↓
[Coda] — 最終ブロック
  ↓
出力
```

更新式：`h_{t+1} = A·h_t + B·e + Transformer(h_t, e)`

毎ループ、入力情報を注入しながら内部状態（`h_t`）を精緻化する。重みは共有したまま、繰り返すたびに表現が深まる設計だ。

**ACT（Adaptive Computation Time）** によって、ループ回数は問題の複雑さで動的に変わる。簡単な問いは少ないループで止まり、複雑な問いは上限まで繰り返す。

この設計の主張は「770Mパラメータのループモデルが1.3Bの固定深度モデルと同等の品質を達成する」というものだ。これはOpenMythosの数字であり、Anthropicが検証したものではない。独立した再現を要する主張だ。

## 「関心は高いが使っている話を聞かない」の構造

現時点でClaude MythosとOpenMythosのどちらも「使っている」という話が聞こえない理由は、それぞれ異なる。

**Claude Mythosを使えない理由：**
Anthropicの審査を通過した限られたパートナーのみがアクセス可能。不正アクセス事件も発生しており、管理はさらに厳しくなる方向だ。

**OpenMythosを使えない理由：**
事前学習済み重みが存在しない。アーキテクチャのコードはあっても、「Claude Mythosと同等のモデル」を作るには一から数十億ドル規模の計算資源が必要だ。インストールできるのはPyTorchのコードだけであり、動くモデルではない。

**結果として起きていること：**
Claude Mythosへの関心が高まるにつれ、「再構築した」と主張するOSSが注目を集める。コードをインストールして試してみると、動かない。そこで初めて「これはアーキテクチャの推測実装であり、Claude Mythosそのものではない」と気づく。このサイクルが繰り返されている。

## thought-analyzerとの接点

RDTのAdaptive Haltingが正しいなら、「簡単な問い」と「複雑な問い」ではモデルの計算量が変わることになる。

thought-analyzerの9軸のひとつ `integrative_complexity`（統合的複雑性）は、「複数の視点を同時に持ちながら、それらを組み合わせて考える」傾向を測る軸だ。このスコアが高い人は、自然と「AとBが互いにどう影響し合うか、さらにCが加わるとどうなるか」という構造の問いを作る。こうした問いは、Adaptive Haltingの視点から見ると「まだループが必要」と判断されやすい。

ただし正直に言う。これは「OpenMythosのアーキテクチャが正しい」という仮説の上に、「それがthought-analyzerの軸と対応する」という仮説を重ねた話だ。Claude MythosがRDTかどうか自体が不明である以上、ここから先は検証できない推測の領域に入る。

**TA接続の深さより、この記事で伝えたい本質は別のところにある。**

## 結局、何が言いたいか

Claude Mythosをめぐる状況は、AIの現在地を象徴している。

能力が高すぎて公開できないモデルが実在する。外から見えないから、外から推測しようとする人が出てくる。その推測がOSSになって広まる。コードは動かないが、スターだけが積み上がる。

この「関心と実態のギャップ」は、Claude Mythosに限った話ではない。高性能なAIほど、実際に使っている人は少ない。使える人が限られているか、使いこなす前提条件が高すぎるか、どちらかだ。

OpenMythosのコードは「Claude Mythosのアーキテクチャかもしれないもの」の設計図だ。そこから学べることはある。ただし、設計図を持っていることと、建物の中に入れることは別の話だ。

**モデルの能力は、アクセスできなければ意味がない。そしてアクセスできたとして、それを活かせる問いを作れるかどうかが次の問題になる。** thought-analyzerが測ろうとしているのは、後者の部分だ。
